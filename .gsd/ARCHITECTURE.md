# Architecture

> Generated by /map on 2026-06-10

## Overview
A Premium SaaS Cafe Management System with a React frontend and an Express Node.js backend. The application provides a customer-facing UI for browsing the menu and placing orders, and a staff-facing backend for real-time order management.

## System Diagram
```
[Frontend (Vite + React)] <---HTTP/REST---> [Backend (Express API)]
      |                                           |
      |                                     [In-Memory DB]
      |                                      (menu, orders)
      v
 [Client UI: Home, Menu, Cart, Admin]
```

## Components

### Frontend Application
- **Purpose:** Customer-facing web UI and Staff Admin dashboard
- **Location:** `frontend/`
- **Dependencies:** React, React Router, Tailwind CSS, Framer Motion
- **Dependents:** None

### Backend Server
- **Purpose:** API server handling menu items and order creation/management
- **Location:** `backend/`
- **Dependencies:** Express, CORS
- **Dependents:** Frontend Application

## Data Flow
1. User interacts with the Frontend to view Menu items or place an Order.
2. Frontend sends HTTP requests to Backend (`/api/menu`, `/api/orders`).
3. Backend processes the requests and updates the In-Memory Database (`menuItems`, `orders`).
4. Responses are sent back to the Frontend to update UI state.

## Conventions
- **Naming:** CamelCase for variables, PascalCase for React components.
- **Structure:** Monorepo with separated `frontend` and `backend` directories.
- **Testing:** No explicit testing framework currently configured.

## Technical Debt
- [ ] Backend uses an in-memory database (`orders`, `menuItems`), which will reset on server restart. Needs persistent DB.
- [ ] No Websocket implementation yet (PRD specifies real-time Socket.io syncing).
- [ ] Hardcoded ports and API URLs (e.g., `PORT = process.env.PORT || 5001`).
